🚀 Spec Kit — Hướng dẫn Setup từng bước

Bộ công cụ spec-driven (plan → code → review → test → evidence) chạy trên Claude Code, port từ extension Auto Spec Kit (vốn dùng GitHub Copilot). Hướng dẫn này cấu hình ở chế độ chỉ-mình-bạn: chạy lệnh gì cũng không để lại dấu vết trong repo team/dự án.

🔧 Tool: cài 1 lần (global) 📚 KB: riêng từng project 🙈 Zero footprint trong repo team 🔑 Không cần API key
✅ Trạng thái trên máy này

Mọi thứ trong hướng dẫn này đã được cài sẵn trên máy của bạn rồi (symlink vào ~/.claude + global gitignore). Các bước bên dưới giúp bạn (1) hiểu đã setup gì, (2) kiểm tra lại, (3) làm lại trên máy khác nếu cần.

🧭 1. Mô hình hoạt động (đọc 1 phút là hiểu)

🛠️ 2. Các bước Setup

Trên máy hiện tại đã xong. Dùng các bước này khi cài trên máy mới, hoặc để kiểm tra lại.

1

Lấy mã nguồn của tool

Clone repo về một chỗ cố định (đã có sẵn tại /Users/MAC/AI-TOOL/claude-skill trên máy này).

# SSH (khuyên dùng nếu GitHub của bạn xài SSH key)
git clone git@github.com:NamHT4Devlop/claude-skill.git ~/claude-skill

Bên dưới gọi thư mục này là <TOOL>. Trên máy này <TOOL> = /Users/MAC/AI-TOOL/claude-skill.

2

Cài tool cho riêng bạn (symlink vào ~/.claude)

Một lệnh duy nhất — tạo symlink 8 skills + 7 agents + 9 commands vào home của bạn. Lệnh được đặt tiền tố spec- để không đè lệnh built-in (như /help).

/Users/MAC/AI-TOOL/claude-skill/scripts/personal-install.sh

Vì là symlink → sau này git pull trong <TOOL> là tự cập nhật, không phải cài lại.

3

Chặn rò rỉ — global gitignore (cấu hình máy, không đụng repo)

Để dù bạn lỡ chạy lệnh trong repo team, KB & session sinh ra vẫn local + không bao giờ commit.

touch ~/.gitignore_global
printf '%s\n' 'spec-kit-sessions/' 'knowledge-base/' 'CLAUDE.local.md' '.spec-kit/' >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global
4

Kiểm tra

Xác nhận symlink & gitignore hoạt động:

# thấy các symlink spec-* trỏ về <TOOL>
ls -la ~/.claude/skills | grep spec-
# xác nhận KB/sessions bị ignore ở mọi repo
git config --global core.excludesfile

Rồi mở Claude Code, gõ / và xem có /spec-build, /spec-ask… xuất hiện không.

⚠️ Chỉ chọn 1 cách cài

Đã dùng personal-install.sh thì ĐỪNG chạy thêm /plugin install cho cùng plugin này — sẽ bị trùng lệnh (định nghĩa 2 lần).

▶️ 3. Cách dùng hằng ngày

Quan trọng: luôn mở Claude Code bên trong project bạn muốn làm, để nó đọc đúng KB của project đó.

# 1) vào đúng project
cd /Users/MAC/AI-TOOL/human-essentials
claude

# 2) lần đầu với 1 project: tạo Knowledge Base
/spec-scan

# 3) sau đó dùng thoải mái
/spec-ask   luồng thanh toán hoạt động thế nào?
/spec-build thêm tính năng quên mật khẩu qua email OTP, hết hạn sau 10 phút
/spec-review
💬 Mẹo

Không bắt buộc gõ slash command — skill tự kích hoạt khi bạn nói tự nhiên, ví dụ "review giúp tôi file này" hay "build tính năng X theo KB".

❓ 8. FAQ

Mỗi project có phải tạo folder .claude không?

Không. Bạn cài 1 lần ở ~/.claude (global) → mọi project dùng được. Chỉ tạo .claude trong từng repo khi bạn cố tình muốn chia sẻ tool kèm repo cho team — mà bạn thì không.

KB cũ tôi đã gen ở máy khác có dùng lại được không?

Được 100%. KB là Markdown thuần trong knowledge-base/; cứ copy về đúng thư mục project là mọi lệnh đọc được ngay — không cần gen lại.

Cài kiểu skill thuần thì có cần copy thư mục resources/ không?

Không. Mỗi skill đã tự chứa thứ nó cần trong references/ riêng. resources/ ở gốc chỉ là bản canonical cho dạng plugin.

Monorepo thì sao?

Ngược lại với workspace nhiều-repo: mở Claude Code ở gốc repo, KB đặt ở gốc với knowledge-base/modules/ (lệnh scan tự sinh per-module).
